# 模块 01：编辑器核心交互

## 1. 模块目标

建立稳定、可预测、高吞吐的高频交互系统，覆盖：

- 指针命中与选择策略
- 选择态状态机
- 手势与键盘协同
- 命令分发与结果回写

## 2. 业务范围

包含：

1. 单选/多选/框选/直接选择
2. hover 与 active 反馈
3. 指针按下/移动/抬起状态流
4. 快捷键动作触发与冲突消解

不包含：

1. 具体图形几何算法实现细节
2. 引擎渲染管线实现

## 3. 关键用户故事

1. 作为设计师，我希望按住 Shift 快速追加/取消选择而不丢失当前工作上下文。
2. 作为设计师，我希望框选命中的规则可预期（包含/相交）。
3. 作为设计师，我希望 dselector 能精确选择路径锚点和段，避免误命中整体对象。
4. 作为设计师，我希望快捷键与鼠标动作组合时行为一致，不出现随机状态跳变。

## 4. 需求条目

### R1 命中与选择策略统一

1. 所有工具共享统一的命中优先级配置（对象、控制点、叠加层）。
2. 选择命中需要支持上下文参数：

- 编辑模式
- 锁定/隐藏过滤
- 隔离模式范围

验收标准：

- 在相同上下文下，多次点击同一区域结果一致。
- 切换工具后命中规则不出现不合理漂移。

### R2 选择状态机显式化

状态最少包含：

1. idle
2. hover
3. selecting
4. transforming
5. pathEditing
6. dragging

验收标准：

- 任意时刻状态唯一。
- 状态切换必须由明确事件触发且可回放。

### R3 指针生命周期完整性

1. pointerdown 负责会话初始化。
2. pointermove 只更新预览态，不直接污染已提交文档。
3. pointerup 触发 commit/cancel 并清理临时态。

验收标准：

- 中断事件（mouseleave、ESC）后不遗留脏状态。
- 快速点击与拖拽行为边界清晰。

### R4 快捷键系统一致性

1. 快捷键动作统一映射到命令层。
2. 平台差异（meta/ctrl）需要统一抽象。
3. 快捷键冲突按优先级决策并可配置。

验收标准：

- 核心快捷键在 macOS/Windows 行为等价。
- 冲突场景下触发结果可解释。

## 5. 关键指标

1. 误选率（选择结果偏离用户意图）

- 目标：关键场景下降 40%。

2. 交互中断率（因状态错乱导致撤销）

- 目标：下降 50%。

3. 高频操作完成时长

- 目标：选择+移动组合操作平均时长下降 20%。

## 6. 风险与依赖

1. 风险

- 命中逻辑分散导致规则碎片化。

2. 依赖

- Runtime 命中与变换会话稳定输出。
- Overlay 模型可提供可预测反馈。
