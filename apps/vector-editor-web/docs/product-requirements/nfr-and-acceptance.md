# 非功能需求与统一验收

## 1. 性能（Performance）

### NFR-P1 高频交互帧稳定性

要求：

1. 高频交互（拖拽/缩放/旋转/路径手柄）保持稳定响应。
2. 典型文档规模下不得出现连续明显掉帧。

验收：

- 建立基准场景（小/中/大三档文档）并持续回归。

### NFR-P2 命令时延

要求：

1. 指针事件到预览更新延迟可控。
2. 提交操作到文档落地延迟可控。

验收：

- 记录 P50/P95 延迟并纳入构建报告。

## 2. 可靠性（Reliability）

### NFR-R1 状态一致性

要求：

1. 预览态与提交态一致。
2. 撤销/重做与文档状态一致。

验收：

- 自动化回放测试通过。
- 异常中断后不留悬空会话。

### NFR-R2 容错恢复

要求：

1. 运行时异常不得破坏文档主数据。
2. 失败命令应可回滚或忽略。

验收：

- 故障注入测试通过。

## 3. 可用性（Usability）

### NFR-U1 一致性

要求：

1. 工具行为、快捷键语义、面板反馈一致。
2. 相同任务在不同入口结果一致。

验收：

- 关键任务走查（10 条）通过率 >= 90%。

### NFR-U2 可学习性

要求：

1. 新用户能在短时间完成基础作图、编辑、导出。

验收：

- 新手任务完成率与时长满足目标。

## 4. 可观测性（Observability）

### NFR-O1 指标采集

要求：

1. 采集交互耗时、命中预算、提交结果、异常事件。

验收：

- 指标完整率 >= 95%。

### NFR-O2 问题定位

要求：

1. 提供诊断日志与最小复现上下文。

验收：

- 线上问题首轮定位时间显著降低。

## 5. 可维护性（Maintainability）

### NFR-M1 模块边界

要求：

1. Product/Runtime/Engine 边界清晰。
2. 禁止跨层语义污染。

验收：

- 边界扫描规则与 code review 检查项长期启用。

### NFR-M2 文档同步

要求：

1. 需求变更需同步 PRD 与验收条目。

验收：

- 每个迭代均有文档状态更新记录。

## 6. 统一验收清单（跨模块）

1. 需求可达：每条需求有明确用户路径。
2. 验收可测：每条需求至少 1 条验收用例。
3. 性能可证：关键路径有指标与基线。
4. 异常可控：有失败处理策略。
5. 回归可跑：具备自动化或半自动回归脚本。
