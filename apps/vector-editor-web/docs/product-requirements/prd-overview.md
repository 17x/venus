# PRD 总览（Vector 2D 编辑器）

## 1. 产品定位

Vector 是一款面向专业设计与产品设计场景的 2D 矢量编辑器。

目标体验：

- 在高频交互流畅度上接近 Figma。
- 在图形编辑深度上逐步接近 Adobe Illustrator。
- 保持 React 外壳 UI + Product 高频交互机制的双层架构。

## 2. 目标用户

1. UI/UX 设计师

- 关注布局、组件复用、协作与交付效率。

2. 品牌与视觉设计师

- 关注路径编辑、布尔运算、文字排版、导出质量。

3. 前端/设计工程师

- 关注设计稿结构化、可追踪、可转换。

## 3. 产品目标（12 个月）

1. 建立稳定的高频交互主链路

- 选择 -> 变换 -> 样式编辑 -> 历史 -> 导出，形成闭环。

2. 建立可扩展的编辑语义层

- Product 只负责语义与策略，Runtime/Engine 提供能力。

3. 建立可规模化的功能矩阵

- 在不牺牲性能的前提下扩展组件、协作、插件能力。

## 4. 当前能力基线（基于现有 product 逻辑）

已具备：

1. 选择与多选、框选、直接选择（dselector）
2. 移动/缩放/旋转、吸附辅助线
3. 路径锚点与控制柄编辑
4. 编组/解组、蒙版、基础布尔运算
5. 填充/描边/阴影/文字基础样式
6. 图层面板与属性面板
7. 快捷键与撤销重做

## 5. 核心缺口（对标 AI/Figma）

P0：

1. 组件/符号系统（主组件、实例、覆盖）
2. 约束与自动布局
3. 协作模型（多人实时、presence、冲突策略）
4. 高级吸附与栅格/参考线控制面板

P1：

1. 文字高级排版（段落、样式集、文字沿路径）
2. 导出管线增强（SVG/批量/切片）
3. 资产库与样式库
4. 交互原型（链接、转场、交互状态）

P2：

1. 插件扩展 API
2. 高阶路径能力（更多节点类型、复合路径）
3. 可视化性能诊断工具

## 6. 需求分层原则

1. Product 层

- 只定义用户语义、交互策略、状态编排与验收规则。

2. Runtime 层

- 提供命中、变换、叠加层、命令调度等运行态能力。

3. Engine 层

- 提供底层渲染与运行能力，不承载产品语义。

## 6.1 架构决策（2026-05-21）

决策：

1. Product 与 Runtime 暂不合并。

原因：

1. Product 与 Runtime 职责不同：前者负责产品语义，后者负责运行机制。
2. 高频交互链路要求 Runtime 机制稳定，避免与 Product 快速迭代互相牵连。
3. 后续组件、协作、插件能力需要清晰边界支撑扩展。

执行约束：

1. 允许做能力下沉与接口收敛，不做语义层与机制层的物理揉合。
2. 以命令契约和桥接接口对齐，而非目录层面强制合并。

## 7. 版本策略

1. 先补齐主链路稳定性（P0）
2. 再补齐专业能力深度（P1）
3. 最后开放生态能力（P2）

详见 release-plan.md。

## 8. 总体验收口径

1. 功能可达

- 关键任务完成路径不超过 3 步中断。

2. 交互可预期

- 高频操作无明显跳变、误选、状态错乱。

3. 性能可控

- 常见设计文件下保持稳定帧率与可接受延迟。

4. 文档可执行

- 每项需求都有明确验收与测试入口。
