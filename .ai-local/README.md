# AI Local Governance

This directory owns governance documents with project-local semantics.

Positioning:

- Root .ai defines cross-project highest-level governance.
- .ai-local defines Venus-specific enforcement and operational governance.
- Historical AI archive content has been consolidated under .ai-local/archive/.

Primary local governance files:

- AI_HIGHEST_STANDARD.md
- workflow.md
- module-map.md
- commands.md
- rules-index.md
- search-playbook.md
- refactor.md
- scattered-rules-consolidated.md
- draft.md

Task-doc separation:

- Running task documents are stored in .ai-tasks/.
- .ai-local remains for governance and execution policy, not task tracking.
