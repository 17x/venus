# 资产压缩基线

## 目的

定义面向引擎演进的、语义中立且 API-first 的压缩能力基线。

## 压缩类别

- 几何压缩
- 纹理压缩
- 动画压缩
- 体数据压缩
- 流式分块压缩
- 传输层压缩感知

## 分层归属

- kernel：压缩感知资产契约与解码状态模型
- orchestration：解码/转码调度与生命周期编排
- optimization：精度、延迟、压力策略仲裁
- platform：按宿主运行时实现 codec/transcoder 适配
- backend：GPU 压缩格式能力协商与上传回退

## API-First 门禁

- 压缩能力必须通过受治理 runtime API 暴露。
- 压缩标识中禁止产品/行业语义。
- 禁止通过深层模块路径直接接入压缩流程。

## 3D-First 门禁

- 压缩基线优先服务 3D 运行时。
- 2D 特定压缩流程必须显式 opt-in 且保持隔离。

## 首批能力目标

- 压缩资产注册与描述契约
- 异步解码状态与诊断可见性
- 后端格式探测与回退轨迹
- 帧预算感知的解码调度
- Node/无头运行时确定性解码检查点

## 当前契约对齐（DEX-034/035/036）

- 资源描述契约已暴露几何/动画压缩策略载荷，包含量化、增量模式、分块大小与检查点策略。
- 运行时驻留输出已暴露确定性解码精度与检查点模式字段。
- 后端诊断已暴露压缩纹理支持、上传路径决策（`direct|transcode|uncompressed`）与回退原因。
- WebGPU 压缩格式协商支持基于适配器探测载荷；当探测缺失时保持确定性的未压缩回退路径。

## 描述符治理检查清单

- 每次新增压缩描述符字段必须在同一切片同步 EN/CN 文档。
- 运行时 hard-cut 契约测试必须覆盖压缩注册、更新迁移、解压回退重置行为。
- 当描述符语义对外可观测时，必须同步 capability-map 与 API 治理文档。
