# 场景能力矩阵

## 目的

该矩阵定义 13 个目标场景如何由通用引擎运行时能力组合而成，且不引入行业产品语义。

硬约束：

- 默认运行时保持 3D-first
- 2D 仅允许显式 opt-in
- 语义中立（不承载业务语义）
- API-first 暴露，命名简洁

## 能力域

- C1：场景与图运行时
- C2：空间查询与拾取
- C3：相机与视图编排
- C4：渲染规划与提取
- C5：GPU 执行与后端回退
- C6：资源图与驻留
- C7：流式加载与渐进细化
- C8：帧预算与压力仲裁
- C9：交互与工具运行时
- C10：时间线与回放运行时
- C11：体数据与三维场运行时
- C12：多视口与合成表面
- C13：平台与无头适配
- C14：可观测性与诊断
- C15：2D 显式模块组（可选）

## 分层 Owner 图例

- B：backend
- K：kernel
- O：optimization
- R：orchestration
- P：platform

## API 暴露规则

- 每个能力域必须先映射到受治理的 engine API。
- 任一场景不得依赖 API 之外的深层模块导入。
- API 命名必须简洁且保持场景语义中立。
- 新增能力域必须先完成 B K O R P 归属论证，再进入实现。
